Fundamentos de Redes de Computadores

Conceitos | Protocolos | Arquitetura | Aplicações

As redes de computadores são a espinha dorsal da comunicação digital moderna, conectando dispositivos e pessoas em todo o mundo. Elas permitem o compartilhamento de recursos, troca de informações e acesso a serviços globais como internet, email e streaming.

Este campo do conhecimento envolve conceitos como protocolos de comunicação, topologias de rede, hardware de conectividade e segurança digital, servindo como base para administradores de rede, desenvolvedores e profissionais de TI.

O que são Redes de Computadores?

Redes de Computadores são conjuntos de dois ou mais dispositivos interconectados que compartilham recursos e trocam informações. Esses dispositivos podem ser computadores, smartphones, servidores, roteadores, switches e qualquer equipamento capaz de se comunicar digitalmente.

As redes permitem que usuários compartilhem arquivos, impressoras, acesso à internet e executem aplicações colaborativas. Elas podem ser classificadas por tamanho (LAN, MAN, WAN), meio de transmissão (cabeadas ou wireless) e topologia (estrela, anel, barramento).

O que é uma Rede Mínima?

Uma rede mínima é a configuração mais simples possível para conectar dispositivos. Basicamente consiste em:

🔗 Elementos Essenciais
  • Dois dispositivos - computadores, celulares, etc.
  • Meio de conexão - cabo ou sinal wireless
  • Protocolo de comunicação - regras para troca de dados

Exemplo: Conexão ponto a ponto entre dois computadores usando um cabo crossover ou uma conexão Wi-Fi direta ad-hoc.

Protocolos de Comunicação

Exemplificação | Funcionalidade | Utilização

Um protocolo de comunicação é um conjunto de regras que define como os dispositivos trocam informações em uma rede.

Os protocolos especificam formato, sequência de mensagens, detecção de erros e métodos de endereçamento.

Protocolos IP V4 e IP V6

4️⃣

IP V4

  • Uso de NAT
  • Formato: 192.168.0.1

IP V6

  • 128 bits
  • 3.4×10³⁸ endereços
  • IPsec nativo
  • Formato: 2001:db8::1

Configuração IP/TCP em Node.js

No Node.js, IP e TCP são configurados com o módulo net para conexões raw.

📄 Servidor TCP em Node.js
const net = require('net');
const server = net.createServer((socket) => {
    console.log('Cliente conectado: ' + socket.remoteAddress);
    socket.write('Bem-vindo ao servidor TCP!\r\n');
    socket.on('data', (data) => {
        console.log('Dados recebidos: ' + data);
    });
});

server.listen(8080, '0.0.0.0', () => {
    console.log('Servidor TCP ouvindo na porta 8080');
});

Protocolo HTTP

HTTP (HyperText Transfer Protocol) é o protocolo de aplicação para transferência de hipertexto na Web.

📋 Características
  • Stateless - Cada requisição é independente
  • Porta 80 - Porta padrão
  • Métodos: GET, POST, PUT, DELETE
  • Códigos: 200 (OK), 404 (Não encontrado), 500 (Erro interno)

Protocolo HTTPS

HTTPS é a versão segura do HTTP utilizando TLS/SSL para criptografia.

Troca de Mensagens Frontend/Backend com Arduino

📄 API REST com Express.js + Comunicação Serial Arduino
const express = require('express')
const app = express()
const { SerialPort, ReadlineParser } = require('serialport')
const readline = require('readline')
const cors = require('cors')

const PORT = 3000
const hostname = 'localhost'
const BAUD_RATE = 9600
const SERIAL_PORT = "COM9"

app.use(express.urlencoded({extended: true}))
app.use(express.json())
app.use(cors())

const arduino = new SerialPort({
    path: SERIAL_PORT,
    baudRate: BAUD_RATE,
})

const parserDados = arduino.pipe(new ReadlineParser({delimiter: "\r\n"}))

arduino.on("open", ()=>{
    console.log(`Conectado ao Arduino em ${SERIAL_PORT}`)
})

arduino.on("error", (err)=>{
    console.error('Erro na comunicação serial', err)
})

parserDados.on("data", (dad)=>{
    const linha = dad.toString().trim()
    if(linha) console.log('Arduino: ', linha)
})
