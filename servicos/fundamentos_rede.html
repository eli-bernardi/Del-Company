Fundamentos de Redes de Computadores

Conceitos | Protocolos | Arquitetura | Aplicações

As redes de computadores são a espinha dorsal da comunicação digital moderna, conectando dispositivos e pessoas em todo o mundo. Elas permitem o compartilhamento de recursos, troca de informações e acesso a serviços globais como internet, email e streaming.

Este campo do conhecimento envolve conceitos como protocolos de comunicação, topologias de rede, hardware de conectividade e segurança digital, servindo como base para administradores de rede, desenvolvedores e profissionais de TI.


O que são Redes de Computadores?

Redes de Computadores são conjuntos de dois ou mais dispositivos interconectados que compartilham recursos e trocam informações. Esses dispositivos podem ser computadores, smartphones, servidores, roteadores, switches e qualquer equipamento capaz de se comunicar digitalmente.

As redes permitem que usuários compartilhem arquivos, impressoras, acesso à internet e executem aplicações colaborativas. Elas podem ser classificadas por tamanho (LAN, MAN, WAN), meio de transmissão (cabeadas ou wireless) e topologia (estrela, anel, barramento).

O que é uma Rede Mínima?

Uma rede mínima é a configuração mais simples possível para conectar dispositivos. Basicamente consiste em:

  • Dois dispositivos (dois computadores ou um computador e uma impressora)
  • Meio de conexão (cabo Ethernet ou conexão wireless)
  • Protocolo de comunicação (TCP/IP básico)

Exemplo: Conexão ponto a ponto entre dois computadores usando um cabo crossover ou uma conexão Wi-Fi direta (ad-hoc).

O que é a Internet? O que a compõe?

A Internet é uma rede global de redes interconectadas que utiliza o protocolo TCP/IP para comunicação. É composta por:

  • Backbone - Redes tronco de alta capacidade
  • Provedores de Serviço Internet (ISPs)
  • Data Centers e servidores
  • Roteadores e switches
  • Cabos submarinos e links de satélite
  • Dispositivos dos usuários finais

O que é a Web?

A Web (World Wide Web) é um sistema de informações que funciona sobre a Internet, composto por documentos interligados (páginas web) acessíveis através de navegadores. Diferente da Internet (a infraestrutura), a Web é o conteúdo e serviços acessados através dela.

Protocolos de Comunicação

Exemplificação | Funcionalidade | Utilização

Um protocolo de comunicação é um conjunto de regras e padrões que define como os dispositivos em uma rede trocam informações. É como uma "língua comum" que todos os dispositivos entendem para se comunicarem eficientemente.

Os protocolos especificam formato dos dados, sequência de mensagens, detecção e correção de erros, controle de fluxo e métodos de endereçamento.

Modelo OSI

O Modelo OSI (Open Systems Interconnection) é um modelo de referência que divide a comunicação de rede em 7 camadas:

  1. Física - Meio físico de transmissão
  2. Enlace de Dados - Controle de erros no link
  3. Rede - Roteamento e endereçamento lógico
  4. Transporte - Controle de fluxo e confiabilidade
  5. Sessão - Estabelecimento de sessões
  6. Apresentação - Formatação e criptografia de dados
  7. Aplicação - Interface com usuário final

Modelo TCP/IP

O Modelo TCP/IP é o modelo prático usado na Internet, com 4 camadas:

  1. Aplicação - HTTP, FTP, SMTP, DNS
  2. Transporte - TCP, UDP
  3. Internet - IP, ICMP
  4. Interface de Rede - Ethernet, Wi-Fi

Protocolos IP V4 e IP V6

IP V4 (Internet Protocol version 4):

  • Endereços de 32 bits (ex: 192.168.1.1)
  • ≈ 4.3 bilhões de endereços possíveis
  • Esgotado devido ao crescimento da Internet
  • Usa NAT para extensão de endereços

IP V6 (Internet Protocol version 6):

  • Endereços de 128 bits (ex: 2001:0db8:85a3:0000:0000:8a2e:0370:7334)
  • ≈ 3.4×10³⁸ endereços possíveis
  • Segurança nativa (IPsec)
  • Melhor eficiência no roteamento

Configuração IP/TCP em Node.js

No Node.js, a configuração de IP e TCP é feita através do módulo net para conexões raw ou http/https para aplicações web.

Exemplo de servidor TCP em Node.js:

                        
                            // Cria um servidor TCP simples
                            const net = require('net');
                            const server = net.createServer((socket) => {
                                console.log('Cliente conectado: ' + socket.remoteAddress);
                                socket.write('Bem-vindo ao servidor TCP!\r\n');
                                socket.on('data', (data) => {
                                    console.log('Dados recebidos: ' + data);
                                });
                            });


                            server.listen(8080, '0.0.0.0', () => {

                            console.log('Servidor TCP ouvindo na porta 8080');

                            });

Protocolo HTTP

HTTP (HyperText Transfer Protocol) é o protocolo de aplicação para transferência de hipertexto na Web. Opera no modelo cliente-servidor usando requisições e respostas.

Características:

  • Stateless (sem estado)
  • Porta 80 padrão
  • Métodos: GET, POST, PUT, DELETE
  • Códigos de status: 200, 404, 500, etc.

Protocolo HTTPS

HTTPS (HTTP Secure) é a versão segura do HTTP que utiliza criptografia TLS/SSL para proteger a comunicação.

Vantagens:

  • Confidencialidade dos dados
  • Integridade da informação
  • Autenticação do servidor
  • Porta 443 padrão

Exemplo 2: WebSockets com Socket.io

                        
                            const express = require('express');
                            const http = require('http');
                            const socketIo = require('socket.io');
                            
                            const app = express();
                            const server = http.createServer(app);
                            const io = socketIo(server);
                            
                            io.on('connection', (socket) => {
                            console.log('Usuário conectado:', socket.id);
                            
                            socket.on('mensagem', (data) => {
                            console.log('Mensagem:', data);
                            io.emit('nova_mensagem', {
                            usuario: socket.id,
                            texto: data,
                            timestamp: new Date()
                            });
                            });
                            
                            socket.on('disconnect', () => {
                            console.log('Usuário desconectado:', socket.id);
                            });
                            });
                            
                            server.listen(3000, () => {
                            console.log('Servidor Socket.io na porta 3000');
                            });
Exemplo 3: API com Autenticação JWT (JSON WEB TOKENS)

                        
                            const express = require('express');
                            const jwt = require('jsonwebtoken');
                            const app = express();
                            
                            app.use(express.json());
                            
                            const SECRET = 'chave-secreta';
                            
                            // Login
                            app.post('/api/login', (req, res) => {
                            const { usuario, senha } = req.body;
                            if (usuario === 'admin' && senha === '1234') {
                            const token = jwt.sign({ usuario }, SECRET, { expiresIn: '1h' });
                            res.json({ token });
                            } else {
                            res.status(401).json({ erro: 'Credenciais inválidas' });
                            }
                            });
                            
                            // Rota protegida
                            app.get('/api/protegido', (req, res) => {
                            const token = req.headers.authorization?.split(' ')[1];
                            if (!token) return res.status(401).json({ erro: 'Token necessário' });
                            
                            jwt.verify(token, SECRET, (err, decoded) => {
                            if (err) return res.status(401).json({ erro: 'Token inválido' });
                            res.json({ mensagem: 'Acesso permitido!', usuario: decoded.usuario });
                            });
                            });
                            
                            app.listen(3000, () => {
                            console.log('Servidor de autenticação na porta 3000');
                            });
